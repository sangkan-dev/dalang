# Web UI

Dalang includes a built-in web interface that provides a modern, browser-based experience for managing scans, viewing reports, and configuring settings ‚Äî all from a single dashboard.

## Starting the Web Server

```bash
dalang web --port 1337
```

This launches the web server at `http://localhost:1337` and opens your default browser. The UI is embedded directly in the Dalang binary ‚Äî no separate frontend build step is needed.

### Options

| Flag | Default | Description |
|------|---------|-------------|
| `--port` | `1337` | HTTP port to listen on |
| `--no-open` | ‚Äî | Don't auto-open the browser |
| `--verbose` | ‚Äî | Enable verbose engine output |

## Pages

### Chat (Interactive Mode)

The main chat page works like a familiar AI assistant interface:

1. **Create a session** ‚Äî enter the target URL and choose a mode (interactive or scan).
2. **Chat in real-time** ‚Äî messages stream over WebSocket. Tool executions and observations appear inline.
3. **Session history** ‚Äî switch between sessions via the sidebar. Sessions persist across page reloads.

**Keyboard shortcuts:**
- `Enter` ‚Äî send message
- `Shift+Enter` ‚Äî newline
- `Ctrl+N` ‚Äî new session
- `Ctrl+K` ‚Äî command palette

### Skills

Browse the full skill catalog (22+ built-in skills). Features:

- **Search & filter** ‚Äî type to filter skills by name or description.
- **Grid / List toggle** ‚Äî switch between card grid and compact list view.
- **Enable / Disable** ‚Äî toggle individual skills on or off. Disabled skills won't be offered to the LLM engine.
- **Detail view** ‚Äî click a skill to see its system prompt, arguments, constraints, and tool path.

### Reports

View and export pentest reports generated by scan sessions:

- Rendered Markdown with syntax highlighting.
- Download as Markdown or HTML.
- Print-friendly layout via the browser's print dialog.

### Settings

Configure Dalang's LLM connection:

- **Provider** ‚Äî choose between Gemini, OpenAI, Anthropic, or Ollama.
- **Model** ‚Äî select from preset models or enter a custom model name.
- **API Key** ‚Äî enter and securely store your API key (persisted via the OS keyring).
- **Test Connection** ‚Äî verify the LLM responds before starting a scan.
- **Endpoint Mode** ‚Äî OpenAI-compatible or Google REST.
- **Verbose toggle** ‚Äî enable extra debug output in engine responses.

## Architecture

The web server is built with:

- **Backend**: [axum](https://github.com/tokio-rs/axum) 0.8 (Rust) ‚Äî REST API + WebSocket.
- **Frontend**: [Svelte 5](https://svelte.dev/) + Tailwind CSS 4 ‚Äî compiled and embedded in the binary via `rust-embed`.
- **Communication**: REST for CRUD, WebSocket for real-time chat streaming.

See [Web Server Architecture](/architecture/web-server) for a deep dive.
